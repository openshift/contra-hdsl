<p>The configureInfra variable handles configuring infrastructure deployed by the contraDSL by executing Ansible playbooks defined in the configuration yaml file.</p>

<h1 id="requirements">Requirements</h1>

<p>Usage of this variable assumes that infrastructure has been deployed via the use of the <code>deployInfra</code> variable, or that an Ansible inventory file exists in the root of the <code>WORKSPACE</code> for the Jenkins job and is named <code>inventory</code>.</p>

<h1 id="examples">Examples:</h1>

<h2 id="basicusage">Basic usage:</h2>

<pre><code>configureInfra
</code></pre>

<p>or</p>

<pre><code>configureInfra()
</code></pre>

<h2 id="enablingverboseoutput">Enabling verbose output:</h2>

<pre><code>configureInfra verbose: true
</code></pre>

<p>This usage allows for the passing of a specific configuration file. If the file doesn't exist in the root of the <code>WORKSPACE</code>, the path to the configuration file should be relative to the root of <code>WORKSPACE</code> for the Jenkins job.</p>
